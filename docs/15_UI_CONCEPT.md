# UI Concept

AIR4 UI should feel like a personal dashboard, not just chat.

## Main Screens

### 1. Today Feed
Timeline of events from the day.

### 2. Chat
Conversation with AIR4 brain.

### 3. Memory
Searchable archive of events.

### 4. Insights
AI observations and patterns.

### 5. Objects
Projects, habits, goals.

## UX Principles

- Minimal friction for event input
- Natural language first
- Insights > raw data
- AIR4 personality visible in UI

User types:

+ bench press 80x8
+ idea: simplify memory schema
