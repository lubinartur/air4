# AIR4 Core Principles

This document defines the non‑negotiable design rules of AIR4.

## 1. Local‑First Memory
All personal data is stored locally.
Cloud models may assist reasoning but never own the data.

## 2. Single Personality
AIR4 is one intelligence.
Agents are tools, not independent personalities.

## 3. Events Are the Source of Truth
Everything the system knows originates from events.

## 4. Append‑Only History
Events are never deleted or overwritten.
Corrections create new events.

## 5. Meaning Must Be Verifiable
Hypotheses about the user must be confirmed or rejected.

## 6. Observations Must Be Rare
AIR4 should speak only when something meaningful happens.
Maximum: 3–5 observations per week.
